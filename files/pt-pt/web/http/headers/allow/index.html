O cabeçalho Allow lista os métodos que um recurso suporta.

Este cabeçalho deve ser enviado se o servidor responder com um código de estado {{HTTPStatus("405")}} Method Not Allowed para indicar que métodos de pedido podem ser utilizados. Um cabeçalho Allowed vazio indica que o recurso não permite métodos de pedido, que podem ocorrer temporariamente para um dado recurso, por exemplo.

Categoria de cabeçalho {{Glossary("Entity header")}}
{{Glossary("Forbidden header name")}} não

Sintaxe

Allow: <métodos-de-http>

Diretivos

<métodos-de-http>
Uma lista de métodos de pedido HTTP permitidos separados por vírgulas.

Exemplos

Allow: GET, POST, HEAD
