En 200 OK HTTP status svarskod förmedlar att en förfrågan har lyckats. Ett 200 svar är cacheningsbar som standard.

Betydelsen för att en förfrågan har lyckats är olika beroende på vilken HTTP förfrågningsmetod du använder:

  • {{HTTPMethod("GET")}}: Resursen har blivit hämtad och finns med i meddelandekroppen.
  • {{HTTPMethod("HEAD")}}: Enhetsrubrikerna finns i meddelandekroppen.
  • {{HTTPMethod("POST")}}: Resursen som beskriver resultatet för händelsen finns med i meddelandekroppen.
  • {{HTTPMethod("TRACE")}}: Meddelandekroppen innehåller förfrågans meddelande som togs emot av servern.

Resultatet för en lyckad {{HTTPMethod("PUT")}} eller en {{HTTPMethod("DELETE")}} förfrågan är oftast inte 200 OK utan {{HTTPStatus("204")}} No Content (eller {{HTTPStatus("201")}} Created om resursen laddades upp för första gången).
